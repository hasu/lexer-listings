6.3

Lexer Listings

Tero Hasu

This library implements something similar to scribble/manual module’s codeblock, codeblock0, code, and typeset-code. However, the typesetting forms and functions here assume foreign code, and no #lang to specify the language used. Instead, one must explicitly specify a color lexer function to use, one with the same signature as that of default-lexer and racket-lexer.

This package presently includes only one lexer, namely the cxx-lexer in the lexer-listings/cxx module.

 (require lexer-listings) package: lexer-listings

The lexcodeblock, lexcodeblock0, lexcode, and typeset-lexcode forms are used for typesetting foreign code, with a lexer explicitly specified as the first argument. The styling of 'keyword tokens can be specified via the current-lexcode-keyword-style parameter.

syntax

(lexcode lexer-expr str-expr ...+)

 
  str-expr : string?
Similar to code, but parses code from strings into the inline text of the document using the specified lexer.

For example,

This is @lexcode[cxx-lexer]{1 + 2}.

produces the typeset result:

This is 1 + 2.

str-expr is a list of strings representing code.

syntax

(lexcodeblock lexer-expr str-expr ...+)

 
  str-expr : string?
Similar to codeblock, but parses code from strings into a block in the document using the specified lexer.

For example,

@lexcodeblock[cxx-lexer]|{
  int f(int x) {
    if (f <= 1) return 1;
    else        return x*f(x-1);
  }
}|

produces the typeset result:

int f(int x) {
  if (f <= 1) return 1;
  else        return x*f(x-1);
}

str-expr is a list of strings representing code.

syntax

(lexcodeblock0 lexer-expr option ... str-expr ...+)

Similar to codeblock0, but uses the specified lexer.

procedure

(typeset-lexcode strs ...) → block?

  strs : string?
Similar to typeset-lexcode, but uses the specified lexer.

parameter

(current-lexcode-keyword-style) → element-style?

(current-lexcode-keyword-style style) → void?
  style : element-style?
A parameter that controls the style used to render symbols that are keywords or perhaps otherwise predefined names. The default is to use keyword-color (mapping to RktKw in LaTeX).

One might also consider value-link-color (or RktValLink), to make known names look similar to for-label defined names in Racket listings (as is configured for this document). The RktValLink choice could cause confusion, however, since symbols so styled look like links, but do not get linked to anything.

See Manual CSS Style Classes for a list of Racket manual styles you might use, or come up with your own alternative style? definition.
